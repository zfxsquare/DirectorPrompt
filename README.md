## 核心体验

DirectorPrompt 是一款 AI 交互式叙事桌面软件，核心定位是**导演模式**：用户不扮演任何角色，不出现在剧情中，而是作为导演通过指令推动整个世界运行，AI 负责生成叙事内容。

与市面上主流的角色扮演类工具不同，DirectorPrompt 为导演模式从零搭建了一整套支撑系统 —— 世界知识管理、故事记忆召回、人物状态追踪、叙事一致性审计、时间线管理。

## 快速开始

### 环境要求

- Windows 11
- .NET 10 SDK
- （可选）Ollama 用于本地模型

### 构建

```bash
dotnet build DirectorPrompt.slnx
```

### 运行

```bash
dotnet run --project DirectorPrompt
```

### 首次使用

1. 启动后进入设置，配置 AI 接口，配置 Embedding 模型
2. 创建项目
3. 编写项目提示词、填充知识条目、设定任务分类、规定状态系统
5. 开始游玩

## 差异

| | 角色扮演工具 | DirectorPrompt |
|--|-------------|----------------|
| 用户身份 | 扮演特定角色参与剧情 | 导演，调度剧情框架 |
| 记忆管理 | 无系统级记忆，靠上下文硬塞 | 向量语义召回 + 时间衰减 |
| 知识触发 | 关键词匹配 | 语义 + 实体混合检索 |
| 状态系统 | 无 | 全局状态 + 人物状态，阶段驱动叙事 |
| 审计机制 | 无 | 分维度并行审计 |
| 时间线 | 无 | 一维坐标轴，支持闪回 |
| 架构 | 前端脚本 | 分层领域模型，可测试可扩展 |

## 功能

### 导演指令组合器

底部输入区并非简单聊天框，而是支持多指令组合、排序、分类选择的指令组合器：

- **剧情指令** — 直接注入叙事事件，一次性消费
- **基调指令** — 约束叙事风格，支持 TTL 自动过期
- **临时约束** — 短期硬性规则，可手动撤销或到期移除
- **场景变更** — 时间跳跃、地点切换

指令排列顺序本身携带语义。先写"三天后"再写"主角发现密信"，与顺序反过来，AI 的理解完全不同。

### 实时状态栏

侧边栏状态栏实时反映世界运转。支持自定义任意状态：金钱、声望、理智值、天气、政治局势、今日任务等。状态支持三种形态（数值、选项、任务列表）和两种驱动方式（AI 从叙事中提取变更，或系统按规则自动变换）。

**阶段效果**是状态系统的核心机制：状态值跨过阈值时自动触发连锁反应。例如理智值跌破 30，世界自动进入"精神崩溃"阶段，相关隐藏设定被激活，叙事基调自动转向压抑混乱。无需手动编写规则，系统在后台处理。

### 语义知识召回

传统工具依赖关键词匹配触发世界设定（如提到"剑"才注入剑术规则）。DirectorPrompt 采用向量语义检索 —— 提到"窄道"也能召回"巷子"相关设定，因为 AI 理解语义关联。同时保留实体精确匹配，确保提到具体人名时该人物背景绝不遗漏。

世界设定支持分组管理。可将后期才揭示的设定放入独立分组，前期关闭，时机成熟后手动开启。叙诡、反转、世界观展开的节奏完全可控。

### 记忆与遗忘

故事越长，历史越难全部塞进 AI 上下文。DirectorPrompt 自动将旧场景压缩为记忆摘要存入向量库。召回时按语义相关性和时间衰减双重排序 —— 久远记忆权重自然降低，但不会被删除，偶尔仍能被相关话题唤起。

记忆归属时间线，闪回场景自动只能看到该时间点之前的记忆，无需手动标记有效性。

### 人物关系网络

人物并非静态卡片。支持分类（可多重继承）、状态栏（跟随分类走）、有向关系（A 对 B 的仇恨与 B 对 A 的愧疚可以是两回事）。关系变化自动产生记忆，形成人物经历档案。

AI 自动维护人物：新人物登场自动建档，人物离场自动标记状态，关系变化自动更新网络。用户只需发指令，系统负责记账。

### 审计：自动查错

AI 生成叙事时可能出现各类错误：已死人物重新出场、违背世界设定、状态值矛盾、时间线冲突。审计系统自动检测这些问题，按严重程度处理 —— 硬伤可阻断并要求重生成，轻微问题可标记警告后放行。无需逐字逐句人工检查。

### 时间线与闪回

故事时间在内部建模为一维坐标轴。线性推进时自然流畅，闪回只需发一条场景变更指令，AI 自动将新场景插入正确时间位置。闪回中的状态变更不会污染当前时间线，记忆自动过滤，回到主线时无缝衔接。

### MCP 扩展

内置工具集覆盖人物、状态、知识、记忆、场景管理。同时支持 MCP 协议双向集成 —— 既可对外暴露 DirectorPrompt 的能力，也可连接外部 MCP 服务获取更多工具。

### 精确回滚与修正

对任何一轮叙事输出，支持四种操作：

- **删除** — 回滚该轮全部状态变更、记忆更新、人物变动
- **手动编辑** — 直接修改文字，不动系统状态
- **修正** — 指出问题后重走完整流程生成新版本，接受才替换，拒绝保留原样
- **重写** — 用相同指令让 AI 重新发挥

所有变更记录为事件序列，可重放、可回溯。
